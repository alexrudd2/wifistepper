{% mdicon wrench %}Hookup guide

This board is a Wi-Fi module connected to a stepper motor driver. It combines both the driver control logic (such as velocity commands, absolute position commands, etc.) with the current/voltage H-Bridge controller that drives a single stepper motor. To get started, you'll need:

  1. 12V to 80V DC voltage source. You can use a standard laptop power brick with a barrel connector (5.5mm OD x 2.5mm ID center positive) or any power supply that can drive enough current (you probably want at least 2-3 Amps).
  2. Bipolar Stepper Motor. There are many different kinds. For beginners we recommend a NEMA 17 because it's a good tradeoff between size, power, and price. If you use a unipolar stepper motor you'll need to wire the leads so that it is electrically identical to a bipolar motor.

You can use either the screw terminals or barrel jack for voltage input (they are connected in parallel).

There will be four wires (two pairs of wires) from your stepper motor that you'll need to connect to the board. They'll most likely be all different colors, so determining which are the pairs is important. The easiest way is to look at the datasheet that came with your stepper motor. If you don't want to do that, connect an ohm meter to two pairs and test for connectivity. Pairs could read anywhere fom 0 to 10 ohms; non-pairs will read Not Connected or very high. Repeat the test using all combinations of wires until you've found the pairs. Connect both pairs A & B to the screw terminals on the board.

Polarity is not important because the software has a Reverse Wires configuration setting that will reverse the direction in software. (Reverse polarity = reverse direction). If you don't want to use that configuration setting and the motor is spinning backwards, choose either A or B wires (but not both), and reverse the (+) plus and (-) minus colors.

{% mdicon flash %}High Voltage and Current Operation (50 volts and above)

With proper configuration, the Wi-Fi Stepper can handle up to 80 volts input. Board modification is needed when using 50 volts and higher. This is due to two factors: 1) Input ratings on the barrel connector and 2) the maximum input voltage of the DC-DC regulator. When operating at 50+ volts, please ensure the following:

  1. Use only the screw terminals for voltage input. (The barrel connector is not rated for more than 50 volts)
  2. Cut jumper JP0 (located mid-board next to DC-DC regulator) and provide 3.3 volt regulated logic supply separately to one of the inputs labeled 3V3 and GND on either side of the board.

Special Note: This board needs the Vin input supply to be present prior to the 3.3v logic supply being present. Under this modification, please ensure that the logic supply voltage is only active when Vin is active as well.

{% mdicon current-ac %}Voltage and Current Modes

There are two control modes available on the Wi-Fi Stepper:

  • Current Mode. Sense resistors are used to detect and control the amount of current flowing throught the motor coils. You specify the peak current using the KT Vals configuration setting. With this mode, the board is capable of 1/16 microstepping.
  • Voltage Mode. The motor driver ignores the sense resistors and pulses a percentage of Vin voltage to the motor coils according to the KT Vals. This causes quieter and smoother operation and the board is capable of 1/128 microstepping.

The following optional board modifications are available under Voltage Mode:

  • Optional (Voltage Mode): Short the sense resistors on the bottom of the board for more efficient operation. Under Voltage Mode, the sense resistors are not required and will cause some undesirable energy dissipation (and lower torque). This board modification is recommended if this board is to be permanently configured with Voltage Mode.

    When switching back to Current Mode, the solder must be removed. The board will not operate under Current Mode with this modification.

  • Optional (Voltage Mode): Add power supply voltage compensation. This modification is only recommended if the power supply is constant, but has the potential to sag or swell under changing loads.

    1. Solder the appropriate resistor to R1 (0805 package). The value can be found using the formula R1 = Vin * 1091 - 1800. Common power supply resistor values are shipped with the Wi-Fi Stepper (12v, 24v, 48v).
    2. Modify jumper JP4 according to the accompanying figure. The left side should be cut and the right side should be bridged (shorted) with solder. This is needed to remove the Vin sense resistor and include the R1 resistor in the circuitry. Note: Vin sensing will no longer work with this board modification. Remove solder bridge on right and add solder bridge on left of JP4 to revert this modification.
    3. Enable Voltage Mode and select Volt Comp under Advanced Settings in the Quick Start Guide.
{% mdicon lan %}Daisy Chaining

A Wi-Fi Stepper acting as a Master node has the capability to send commands to Slave Wi-Fi Steppers over a daisy-chained high speed serial bus. This configuration allows for one master node and up to 32 slave nodes per network. Each slave node is addressed by its index offset to the master node. The master node is referred to as target 0 and slave nodes start at target 1 ... to target n.

One advantage of a daisy-chain configuration is that the master node can be the sole Wi-Fi network interface to a number of stepper motors. Applications only need to manage one network connection to the Master node. Adding an additional axis to your robot/device is easy and only requires wiring up another slave node.

The wiring for a daisy chain network is as follows:

  1. From the Master node, connect Master0.TX → Slave1.RX
  2. From the first Slave node, connect Slave1.TX → Slave2.RX
  3. Repeat 2. for all slave nodes. Reference figure above.
  4. From the last Slave N node, connect SlaveN.TX → Master0.RX completing the daisy chain
  5. All electrical grounds must be connected together.

By default, all Wi-Fi Steppers are configured to be Slave nodes for convenience. (There is no overhead added to a disconnected daisy chain network.) The Master node needs to be configured on the settings page.

  • When enabled, the Turn off slave wifi configuration settings will disable all Slave Wi-Fi controllers so the Master node is the only one accessible over Wi-Fi.
{% mdicon key %}Crypto Authentication

There is an Atmel ECC508a Crypto Authentication chip on the I2C bus for use with the Low Level Interface. When enabled, commands sent to and from the Wi-Fi Stepper can be authenticated using HMAC signatures against an Auth Key. All keys are locked into the EEPROM on-chip on this processor and cannot be read back out.

Two keys are needed to provision the Crypto Chip for use with HMAC: the Master Key and the Auth Key. The Master Key is permanently written and can never be changed once provisioned. The Auth Key is then locked behind the Master Key and is used with the HMAC signature calculation. This allows the Auth Key to be changed only if the Master Key is known as well. However, only the Auth Key is needed during normal operation to generate the HMAC signatures.

To provision the ECC chip, enter the two keys in the settings page. Remember the Master Key as it is permanent. For more information about this chip, visit wifistepper.com/crypto-authentication.html. For use with the Low Level Interface and example code, visit wifistepper.com/interfaces/low-level.html.

{% mdicon bomb %}Hardware Reset

There is a hard reset button located on the board that will restore the configuration to factory defaults. The hard reset procedure is as follows:

  1. With the board not powered, press and hold the reset button.
  2. Power the board with the button still pressed and hold for 5 seconds.
  3. Release the button. The board will wipe the configuration files and revert to factory defaults.
